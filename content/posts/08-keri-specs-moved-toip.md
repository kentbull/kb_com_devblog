# KERI Specifications have moved to the ToIP Foundation

The KERI protocol specifications have moved! (Updated 4-14-2024)

The [Trust over IP (ToIP) Foundation](https://trustoverip.org/) is now hosting the KERI protocol specifications. See the below list for the new specification links as well as the Github repository links. The recent DID WEBS links are listed as well.

## ToIP Specs

- **KERI** – Key Event Receipt Infrastructure
  [Spec](https://trustoverip.github.io/tswg-keri-specification/) | [Github Repo](https://github.com/trustoverip/tswg-keri-specification)

- **ACDC** – Authentic Chained Data Containers
  [Spec](https://trustoverip.github.io/tswg-acdc-specification/) | [Github Repo](https://github.com/trustoverip/tswg-acdc-specification)

- **SAID** – Self Addressing IDentifier
  [Spec](https://trustoverip.github.io/tswg-said-specification/) | [Github Repo](https://github.com/trustoverip/tswg-said-specification)

- **CESR** – Composable Event Streaming Representation
  [Spec](https://trustoverip.github.io/tswg-cesr-specification/) | [Github Repo](https://github.com/trustoverip/tswg-cesr-specification)

- **Trust Spanning Protocol**
  [Spec](https://trustoverip.github.io/tswg-tsp-specification/) | [Github Repo](https://github.com/trustoverip/tswg-tsp-specification)

- **did:webs method**
  [Spec](https://trustoverip.github.io/tswg-did-method-webs-specification/) | [Github Repo](https://github.com/trustoverip/tswg-did-method-webs-specification)
  - did:keri/did:webs resolver implementation
    [Github Repo](https://github.com/WebOfTrust/did-webs-resolver)

Some of the old, IETF specifications are still publicly available yet have either been rolled in to the ToIP specs or have yet to be combined with the ToIP specs. They are included here for reference.
